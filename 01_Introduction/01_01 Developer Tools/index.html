Intoducción

Todos los navegadores modernos incluyen las conocidas como Herramientas de Desarrollo o Developer Tools, este conjunto de herramientas tienen como objetivo, entre otros, ayudarnos a depurar (debugging), analizar, inspeccionar todo lo relacionado al Front-end: código HTML, CSS, Javascript, Typescript, SASS entre otros. Teniendo en cuenta el objetivo del Taller, se va a hacer mención a lo relacionado a CSS y HTML.

Nota:Para los ejemplos de está sección utilizamos el sitio de Unicef. Un organismo de la Organización de las Naciones Unidas que tiene como objetivo promover la defensa de los derechos de los niños, suplir sus necesidades básicas y contribuir a su desarrollo. El sitio suele actualizarse con regularidad, por lo que las capturas del sitio entre los ejemplos son diferentes.

Abrir o mostrar herramientas de desarrollo

Para ocultar/mostrar las herramientas tenemos 3 opciones: Podemos presionar la tecla F12, el juego de teclas ctrl + mayus + I o acceder desde el navegador al menú de opciones del navegador, generalmente situado en la parte superior derecha de la interfaz del navegador, entre las opciones se encuentra el submenú More Tools o en español Más Herramientas, dentro de él, la opción Herramientas de Desarrollo. Por último es posible ingresar inspeccionando algunos elementos del sitio, para ello, click derecho sobre alguna parte del sitio; se abrirá un menú contextual y en él una lista de opciones debemos seleccionar la opción Inspeccionar.

Pestaña Elementos

La pestaña de Elementos está dividida en 2 secciones, una muestra el HTML y la otra las propiedades CSS aplicadas al elemento seleccionado del primero. También dentro del panel derecho tenemos algunas otras pestañas relacionadas a Javascript.

En el panel izquierdo, área del HTML, podemos visualizar el documento HTML cómo un árbol de nodos donde podemos colapsarlos o expandirlos para mostrar o esconder sus nodos hijos, ir navegando entre ellos, seleccionar alguno para ver las propiedades CSS vinculadas. También teniendo el foco en este panel, presionando ctrl + F podemos buscar una cadena de texto, etiqueta HTML o XPath (sucesión de nodos en árbol para ver el camino de un nodo padre a un hijo, ej: "/html/body/footer").

Sí hacemos click derecho sobre alguno de los nodos HTML aparece una lista desplegable con una serie de acciones para hacer sobre el nodo, en la siguiente imagen se detallan:

  1. Add attribute agrega un nuevo atributo HTML.
  2. Edit attribute edita un atributo HTML.
  3. Edit as HTML permite editar en tiempo real el HTML del lado del cliente.
  4. Duplicate element duplica el elemento seleccionado.
  5. Delete element elimina el elemento HTML del DOM.
  6. Hide element esconde el elemento HTML agregando una clase CSS.
  7. Force State fuerza a que un botón, por ejemplo, tenga los típicos estados de los botones (:hover, focus, etc).
  8. Expand Recursively Expande el nodo seleccionado y todos sus hijos.
  9. Collapse Children Colapsa el nodo seleccionado y todos sus hijos
  10. Capture node screenshot descarga una screenshot de una parte del diseño.
  11. Scroll into view sí el elemento está oculto del scroll navega a él.
  12. Badge settings muestra o esconde unas etiquetas que aparecen en el panel izquierdo relacionadas a ciertas cosas como Flex, Grid.
  13. Store as global variable crea una variable global temporal del elemento HTML seleccionado.

En el panel derecho, donde están los estilos vinculados a ese elemento HTML seleccionado, tenemos ordenados por precedencia, herencia y especificación, algo que vamos a ver más adelante en otro módulo, las reglas CSS divididas en bloque por un borde inferior.

  1. Las reglas CSS aplicadas al elemento HTML seleccionado desde la estructura HTML se muestran una bajo otra. En la parte superior derecha del bloque se puede ver un link con el archivo .css que contiene la regla. Haciendo click podemos navegar hacia el archivo dentro de la pestaña Source y ver la regla concreta.
  2. El bloque de la parte superior es un área donde podemos definir propiedades CSS en línea, escribir en está sección implica que el navegador agregue el atributo styles y dentro de él agregue las propiedades CSS que definimos.
  3. :hov Al momento de inspeccionar un elemento con los estados relacionados con las seudo-clases CSS (:hover, :focus, :active y :visited) este es muy útil. Cuando es presionado despliega una lista de checkbox con las seudo-clases y podemos chequear alguno para mantener el estado sin necesidad del mouse, y así inspeccionar el elemento con algún estado.
  4. El botón con el icono del signo + permite crear una nueva regla a partir del elemento HTML seleccionado, sí hacemos click en la esquina inferior derecha del botón, algo similar a una flecha, vemos un desplegable que nos da opciones para seleccionar en cual archivo podemos agregar la regla o directamente en el.
  5. El botón permite listar y agregar clases a un elemento HTML dinámicamente en el navegador, no es agregado a algún archivo CSS. Equivale a editar el HTML y agregar el atributo class con una clase CSS.
  6. Layout, Diseño Dependiendo del navegador y el idioma lo pueden encontrar con algunos de estos nombres. En esta pestaña se puede ver la estructura de un diseño basado Flexbox o Grid Layout, En Firefox se agrega el Box Model, (imagen 2 y 3).
  7. Computed, Calculado Dependiendo del navegador y el idioma lo pueden encontrar con algunos de estos nombres. En esta pestaña se puede ver todas las propiedades CSS aplicadas al elemento inspeccionado. La diferencia con la pestaña anterior es que en esta se verán los valores definitivos de las propiedades que están afectando al elemento HTML (imagen 1).
  8. Styles, Rules o Estilos: Dependiendo del navegador y el idioma lo pueden encontrar con algunos de estos nombres. En esta se visualizan todos los estilos aplicados al elemento inspeccionado por el navegador, alguna librería de terceros, Bootstrap por ejemplo, y las del equipo de desarrollo de la aplicación.

Detalles la Pestaña Calculada (Computed)

La pestaña Calculado, hace referencia a los valores calculados de algunas propiedades CSS del elemento HTML seleccionado.

El porqué de esta pestaña tiene 2 respuestas :

  • La primera es porque algunas propiedades CSS tienen como valor unidades relativas que representan una referencia, los em, rem, porcentajes por ejemplo. El navegador cada vez que le asignamos una medida relativa hace unos cálculos para resolver qué medida absoluta debe tener el elemento. Para nosotros esa medida es relativa pero el navegador al final del cálculo la pasa a una medida absolutas y lo renderiza en pantalla.
  • La segunda, a una propiedad CSS de un elemento HTML se le puede aplicar varias reglas con la misma propiedad. Esto puede llevar que en la pestaña de Elementos seleccionemos uno y nos encontremos con una cantidad de reglas infernal, difícil de leer y comprender qué valor tiene una propiedad CSS. En la pestaña Calculado siempre nos va a mostrar el valor final.
  1. Las propiedades CSS siempre se muestran en relación a un elemento HTML seleccionado en el área del HTML, la sección izquierda. En la imagen se ve un elemento HTML con un fondo gris, este es el elemento seleccionado.
  2. En la parte superior se puede ver un gráfico con el esquema del modelo de cajas del elemento seleccionado. El elemento seleccionado en este caso no tiene definido propiedades del modelo de caja.
  3. En un color rojizo se pueden ver una lista de las propiedades CSS que están afectando al elemento HTML.
  4. A la izquierda de una propiedad en ese color rojizo, siempre que el valor de esta provenga de una hoja de estilos, vamos a ver la lista de todas las reglas y archivos CSS que están afectando a la propiedad.
  5. En un color rojizo semitransparente, aunque no tengamos ninguna regla o archivo CSS relacionado, vamos a ver las propiedades heredadas de un nodo HTML padre.
  6. Es posible filtrar las propiedades de un elemento HTML desde este campo de búsqueda.
  7. Show all. El control por defecto está inactivo. Por defecto en está pestaña se muestran aquellas propiedades que fueron modificadas en algún archivo de hoja de estilos incluido en el documento HTML. Sí activamos el casillero se verán todas las propiedades CSS del elemento HTML seleccionado con los valores por defecto que implementa el navegador. Ver imagen Show all, ver todas la propiedades de un elemento HTML.
  8. Group. Este casillero cuando está activado agrupa las propiedades relacionadas por secciones. Ver imagen a la derecha Propiedades Agrupadas

Detalles del modo "Device"

  1. Esta es una lista desplegable de algunos dispositivos pre-configurados, podemos incluso agregar dispositivos personalizados a esta lista a nuestro local. Sí seleccionamos responsive, Chrome nos da el control de las dimensiones de la ventana de visualización para que lo redimensionemos a nuestro parecer. En la imagen 3 se puede ver una lista de algunas de los dispositivos preconfigurados
  2. Sí el dispositivo seleccionado es uno de la lista de pre-configurados o personalizados estos campos van a mostrar las dimensiones de él, en cambio, sí tenemos seleccionado responsive va a mostrar las dimensiones y va permitirnos editar estos campos numéricos para modificar las dimensiones píxel a píxel.
  3. Muestra en porcentaje el valor del zoom
  4. DPR(Device Pixel Ratio) es la relación entre los píxeles físicos, propios del hardware, y los píxeles CSS que no necesariamente son la misma cosa.
  5. Device Type: El tipo de dispositivo puede ser: Mobile, Mobile no táctil, Desktop o Desktop táctil.
  6. No throttling. Son una serie de opciones que permiten emular diferentes tipos de calidad de conexión, para activar esta emulación tenemos que seleccionar alguna y refrescar el navegador.
  7. Orientación. Este botón permite alternar entre la emulación de la orientación vertical y horizontal de un dispositivo móvil/tableta.
  8. Estas barras de colores son diferentes bloques de los Medios de Consulta (Media Queries) definidos en alguna hoja de estilos CSS, más adelante cuando veamos el módulo de Responsive Web Design vamos a retomar este tema.
  9. Son unas reglas en píxeles para mostrar el tamaño del dispositivo.
  10. Estas barras aparecen cuando tenemos seleccionado responsive para poder alterar las dimensiones de la ventana.
  11. En el menú de opciones podemos esconder/mostrar algunas opciones anteriores, hacer un screenshot o captura de pantalla de la versión móvil de la aplicación y descargarla. Capture screenshot va hacer una captura de lo que estamos viendo en este momento en la ventana, Capture full size screenshot de todo el documento HTML.

Pestaña Fuentes

En panel izquierdo contiene una lista de todos los archivos fuente que conforman el documento HTML actual, el que estamos viendo en la pantalla. Estos archivos puede ser de tipo imagen (jpg, gif , png o svg), código Javascript, CSS, HTML o archivo de fuente tipográfica.

  1. top representa el marco del documento HTML principal.
  2. El primer nivel, en este caso https://www.unicef.org/, es el origen, el dominio desde donde estamos viendo el documento HTML.
  3. El resto de niveles son recursos que carga el primer nivel, el origen.
  4. Podemos expandir los distintos directorios, recursos y navegar por ellos para investigarlos, si hacemos doble click lo abrimos en el panel derecho y quedará anclado mientras no sea cerrado.
  5. El panel derecho o panel central es el área donde podemos visualizar los recursos cargados cuando los abrimos desde el panel izquierdo, estos se acumularán y cuando no quepan aparecerá un menú con los archivos abiertos ocultos.
  6. Los archivos abiertos quedan en memoria y va a aparecer cada vez que entremos a la pestaña Fuentes.
  7. Botón de formateo de CSS y Javascript. Este botón permite formatear en la ventana un archivo minificado, no lo hace sobre el archivo original, es con fines visuales circunstancialmente.
