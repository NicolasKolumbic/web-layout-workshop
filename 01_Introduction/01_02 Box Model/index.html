Introducción

En los documentos HTML a medida que se van incluyendo contenido, etiquetas o estructuras HTML, estás se van acomodando de izquierda a derecha, de arriba hacia abajo, parece algo obvio para aquello que trabajamos con HTML y CSS hace un tiempo, pero es importante conocer que este es el flujo normal de cómo la información se estructura en HTML, es posible cambiar este flujo en algunos idiomas occidentales como japones, hebreo o arabe donde la lectura es derecha a izquierda.

Cada etiqueta tiene lo que se conoce como Modelo de Caja o Box Model. Está brinda un orden interno de capas con objetivos precisos cada uno. CSS tiene propiedades para manipularlas:

A su vez, cada caja puede contener su propia estructura de cajas más pequeñas

Propiedades para manipular el Modelo de Caja

Márgenes

  • margin-top
  • margin-right
  • margin-bottom
  • margin-left

Bordes

  • border-top-color, border-top-width, border-top-style
  • border-right-color, border-right-width, border-right-style
  • border-bottom-color, border-bottom-width, border-bottom-style
  • border-left-color, border-left-width, border-left-style

Relleno

  • padding-top
  • padding-right
  • padding-bottom
  • padding-left

Contenido

  • width
  • max-width
  • min-width
  • height
  • max-height
  • min-height

Cada caja tiene un modelo como este donde ninguna propiedad es obligatoria, ni siquiera el contenido. Podemos crear un elemento HTML como un div vacío y asignarle solo bordes.

Puede suceder que por algún requisito necesitemos salirnos de ese flujo, CSS tiene propiedades que lo permiten, por nombrar las más utilizadas: position, left, right, bottom, top, float, z-index, clear.

La altura del documento HTML

Es típico, en algunos diseños, que una imagen cubra todo lo alto de la pantalla, e intuitivamente pensamos que con incluir un height:100% al contenedor y definir en la propiedad background-image una imagen bastaría, pero no es así.

Un paréntesis para que quede claro el porqué de esto:

En la Imagen 1 nos encontramos con un documento HTML en blanco, a medida que a este se le incluye contenido, crece hacia abajo (Imagen 2), incrementando la altura del body. Cuando la cantidad de contenido sobrepase el alto de nuestro monitor aparecerá la barra de scroll para poder desplazarnos sobre el documento (Imagen 3).¿Entonces alcanza con agregar height: 100% al body?

La respuesta es no, y tiene lógica. En nuestro monitores tenemos claro la resolución, su ancho y alto en píxeles, en el navegador está determinando por ese mismo menos la barra de scroll y el marco del navegador. En el caso, pensamoslo así. El 100% es el todo, un total, cuando aplicamos height:100%¿A qué total nos estamos refiriendo?. No es alto del monitor, es el contenido. Si tuviéramos una medida que represente el 100%, tendremos un alto total determinado, un límite hasta donde incluir el contenido.
