Areas

Grid Layout Module dispone de grid-area y grid-template-area, 2 propiedades que permiten generar estructuras cómo las que venimos realizando de manera muy sencilla. La primera debe declararse sobre los elementos de grilla y recibe un nombre alfanumérico que será el nombre de esa área representada por un elemento HTML. La segunda se incluye en el contenedor y define la estructura de la grilla por medio de los nombres de áreas asignados a cada elemento de grilla.

Vamos a retomar el ejemplo anterior y utilizar areas para estructurarlo. De momento, aside no tendrá la estructura objetivo.

Comentamos algunos detalles sobre las síntaxis de grid-template-areas, las comillas dobles determinan las filas de una grilla. No es válido declarar las áreas en un solo juego de comillas.

Celdas vacías

Puede ser que necesitemos dentro de la estructuración que algunas celdas queden vacías, para esos casos podemos utilizar un caracter "." (punto) o más.

Las áreas y las líneas de grilla

Cuando utilizamos areas para la propiedad grid-area implicitamente y consecuentemente creamos alias para las líneas de la grilla. Chequeando la siguiente imagen observamos las líneas de grilla tiene asignado unos alias (header-start, navbar-start, footer-start, main-start, navbar-end entre otros).

Lo curioso es que si inspeccionamos algunos de estos elementos, por ejemplo el header, vemos que si expandimos la propiedad shorthand grid-area, está generó para las propiedades relacionadas que representan un alias llamado header. grid-area genera líneas de grilla y concatena a los alias un subfijo para el comienzo, -start, y otro para el final, -end.
