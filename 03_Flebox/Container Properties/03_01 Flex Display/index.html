Introducción

Tiempo atrás crear estructuras complejas con CSS y HTML era un dolor de cabeza, además teníamos que luchar con lo que se comentaba como La guerra de Navegadores, donde cada uno, Internet Explorer, Firefox, Chrome y el resto tenía una interpretación propia de los estándares. Cada diseño web conllevaba unos estilos personalizados para cada motor del navegador. Por suerte esto quedó atrás, y surgieron en los últimos años técnicas que facilitaron el armado de estructuras complejas.

Un problema típico que recuerdo eran aquellos diseños donde teníamos columnas con diferentes alturas, en esos tiempos se utilizaba la propiedad float en abundancia para encolumnar div. La siguiente imagen, por si no queda claro, es un ejemplo de lo que estamos comentando.

El objetivo era tener 3 columnas con la misma altura, para ello utilizábamos la propiedad float, está propiedad rompe el comportamiento del elemento en bloque, afectando al contenedor que se ajusta al alto y ancho del contenido, con lo que las alturas de las columnas eran diferentes en relación al contenido, más adelante veremos que Flexbox resuelve este problema con una pocas líneas.

Introducción a Flexbox

Flexbox Layout Module o solo Flexbox es un módulo que consta de un sistema de propiedades con el objetivo de gestionar cómo distribuir el espacio, alinear el contenido y ordenar visualmente en la pantalla, afectando la apariencia y estirando, encogiendo, reorganizando y alterando elementos dentro del flujo del contenedor. El contenido puede distribuirse a lo largo de un solo eje o envolverse en varias líneas, puede crecer para adaptarse a todo el espacio disponible o reducirse para encajar en su contenedor.

Flexbox funciona en un sistema de ejes. Cuando configuramos un elemento para que se presente como una caja flexible, solo afectará a sus hijos directos, los del primer nivel, y no a los descendientes posteriores. Estos elementos secundarios siguen las reglas de Flexbox para el diseño en lugar de las reglas del modelo de caja estándar (nos referimos los elementos en bloque (display:block), en línea, etc...).

Flexbox es estable en escenarios como el caso de contenido que es generado dinámicamente, es fácil reorganizar visualmente el contenido sin afectar el marcado HTML y la apariencia del contenido puede ser independiente del orden de origen.

Las propiedades CSS column-*, los selectores ::first-line y ::first-letter no se aplican cuando se trata del contenedor flexible y los márgenes por defecto en un diseño flexbox, no colapsan.

La compatibilidad con navegadores es más que aceptable; Internet Explorer 10 con el prefijo -ms-, y a partir de las versiones de Microsoft Edge lo se hace de forma estándar. La imagen siguiente es una captura de la compatibilidad con navegadores del sitio Can I Use (Flexbox Layout)

Estructura básica de Flexbox

Un Contenedor Flexible es la caja generada por un elemento con la propiedad display con el valor flex o inline-flex. Los elementos secundarios en el flujo de un Contenedor Flexible se denominan Elementos Flexibles y se distribuyen utilizando el modelo de distribución flexible. Entonces se puede decir que Flexbox funciona a nivel de contenedor, el comportamiento es trasladado a sus elementos contenidos. El contenedor se puede comportar como un elemento en bloque (display:block) o en línea (display:inline).

Diferencia entre elementos flex, inline-flex y la relación con otros valores de la propiedad display

La diferencia entre ellos radica en cómo se comporta el contenedor. En el caso de los elementos flex, el contenedor no permitirá que otros elementos se acomoden a sus lados, igual que display:block, aunque tenga espacio disponible este ocupará todo el ancho disponible de la pantalla. En cambio inline-flex es más parecido a display:inline-block; los contendores con este tipo de display se pueden acomodar uno al lado de los otros.

Primeros conceptos importantes

El Modelo de Caja de Diseño Flexible (Flexbox Layout Box Model) contiene un esquema sobre algunas reglas que dominan el comportamiento, tiene algunos conceptos importantes:

Los primeros 4 conceptos importantes son: main axis, cross axis, main size y cross size.

main axis
El eje principal de un contenedor flexible es el eje principal a lo largo del cual se colocan los elementos flexibles. Por defecto es horizontal.
cross axis
El eje perpendicular al eje principal se llama eje transversal. Por defecto es el vertical.
main size
El ancho o alto de un contenedor flexible, cualquiera que esté en la dimensión principal, es el tamaño principal de esa caja. Su propiedad de tamaño principal es, por tanto, su propiedad de ancho (width) o alto (height), cualquiera que esté en la dimensión principal. De manera similar, sus propiedades de tamaño principal mínimo y máximo son sus propiedades (min-width) / (max-width) o (min-height) / (max-height), cualquiera que esté en la dimensión principal, y determinan su tamaño.
cross size
El ancho o alto de un contenedor flexible, cualquiera que esté en la dimensión transversal, es el tamaño transversal de esa caja. Su propiedad de tamaño cruzado es, por lo tanto, su propiedad de ancho o alto, cualquiera que esté en la dimensión transversal. Del mismo modo, sus propiedades de tamaño transversal mínimo y máximo son sus propiedades (min-width) / (max-width) o (min-height) / (max-height), lo que se encuentra en la dimensión transversal determina su tamaño transversal.

main axis y cross axis son claros, incluso main size no es complejo de comprender en un ejemplo práctico como el del enlace de este apartado. cross size tiene una complejidad dada por la propiedad siguiente de este Taller. align-items.
