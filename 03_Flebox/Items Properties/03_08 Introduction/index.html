Introducción

En este apartado se detalla todas las propiedades relacionadas a los elementos flexibles individuales, que comentamos en la sección anterior. Según las especificaciones los nodos HTML, un nodo de texto o un contenido generado dinámicamente que sean hijos directos, del primer nivel, de los contenedores flexibles son elementos flexibles. La única excepción a esta regla son aquellos elementos posicionados absolutamente (position:absolute) que quedan fuera del flujo flexible.

Elementos flexibles posicionados absolutamente.

Cuando un contenedor flexible contiene elementos flexibles en posición absoluta (position:absolute), estos salen del flujo normal del documento HTML y del flujo del contenedor flexible, pero aun así heredan, sí están aplicadas, las propiedades de alineación para elementos flexibles (justify-content, align-items); como también aquellas propiedades que comentaremos en breve que tienen efecto de manera individual en cada elemento flexible.

Nodos HTML y nodos de textos como elementos flexibles.

Los nodos de texto dentro de un contenedor flexible que solo contienen espacios en blanco son ignorados, como si no existieran o su propiedad display tuviera el valor none. Cuando se trata de nodos de texto, sí el nodo no está vacío, se envolverá en un elemento flexible anónimo, comportándose como sus elementos flexibles hermanos y heredando todas las propiedades flexibles establecidas por el contenedor flexible.

La propiedad float y clear.

Los elementos flexibles no pueden ser flotados, todo lo relacionado a elementos flotantes como la propiedad float y clear no tiene efecto, pero si cambiará el valor calculado de la propiedad display. Finalmente los elementos flotantes se comportarán como elementos flexibles.
